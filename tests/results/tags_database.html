Taking a Tag or Set of Tags for Testing

The name of this file in your checked out code is geant4/tests/results/tags_database.html.
  • Make sure this html file is the head version (cvs update -d -P -A geant4/tests/results).
  • Inspect the Tags Database Table below.  You will need to edit it, i.e., you will need to edit this html page, either with a plain text editor or with Netscape Composer, or similar tool, and then commit it back to the repository (the AFS web page is then automatically updated).  If CVS informs you that there is a conflict, it means that someone else has taken some action.  You will need to resolve the conflict somehow - perhaps by moving your copy of this file, updating again, and starting again.
  • There are 4 possible statuses for a tag or set of tags:
  • accepted - these have been dealt with and need no further action.
  • rejected - again, no need to take any further action.
  • testing in progress.
  • awaiting testing (signified by a blank entry in the Status column)
  • If no testing is in progress, take the lowest order number (#) (take tags of the same order number together) and mark the status as testing.  Commit this file back to the repository and proceed with testing (go back to procedures page).
  • When testing is finished, mark as accepted or rejected and commit this file back to the repository
