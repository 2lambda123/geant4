Bonsai Database for Tags and Testing

The "Bonsai" System

The bonsai system was intoduced in June 2000 and has been used to track tags and testing since the global tag geant4-01-01-ref-03.
If all is well then tagging in the cvs repository will create an entry in the bonsai database.

  • There are 6 possible statuses for a tag:
  • Internal - if you do not see your tag reach this state in bonsai database please contact the stt team.
  • Proposed for Testing - You may request a transition from Internal to Proposed for Testing (check who can do this) or this may be performed by the stt. This is a good opportunity to enter a description of your tag. This is essential a queue of work for the stt team, howver it may be appropriate to revert to this state if one, but not all, of a set of co-working tags are rejected.
  • Selected for Testing - The tag is being tested. The tag has been selected for testing and will contribute to an sdb file extracted from the bonsai database. There are three directories (prod,dev1 and dev2) in which testing may take place. (You cannot tell from the first stage of the Bonsai database which code is on test in which directories - maybe add to this page later. These was possible from the two rightmost columns of the html file with headings # and Status).
  • Tag Accepted - the tag has passed the stt procedures and no futher action from the author is anticipated. The tag should be included in the next reference tag.
  • Tag Rejected - the tag has failed the stt procedures. This may happen if the author (or one of the required platform compilers) fails to compile the code, build the libraries and executables or unacceptable results are produced when executing the test program suite. The stt team should contact the submitter with further information by mail.
  • Global - the tag is a global reference tag and should conform to Geant4 naming conventions.
